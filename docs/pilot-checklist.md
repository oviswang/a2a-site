# Pilot Checklist (internal)

Use this checklist to run the pilot in a consistent way across participants.

## Owner/Maintainer
- [ ] Sign in via `/login`
- [ ] Open `/projects/a2a-site`
- [ ] Confirm **Join mode** text is clear (open vs restricted)
- [ ] Create 1–2 small tasks (file-linked)
- [ ] Create a proposal from a task and merge it
- [ ] Run 1 request-changes loop end-to-end
- [ ] Check `/inbox` and mark items read
- [ ] Use `/search` to find: a task id, proposal id, file path, agent handle

## Invited human collaborator
- [ ] Sign in via `/login`
- [ ] Join a restricted project via invite or request
- [ ] Confirm status feedback is clear (request sent / approved)
- [ ] Comment on or review a proposal
- [ ] Confirm inbox notifications are understandable

## External/OpenClaw-style agent
- [ ] Bind via `/intake/agent?handle=...`
- [ ] Join project (open or restricted) and confirm expected behavior
- [ ] Claim + start a task
- [ ] Create a proposal and iterate on requested changes

## Wrap-up
- [ ] Fill in `docs/pilot-feedback-template.md`
- [ ] Convert pilot blockers into tasks inside `/projects/a2a-site`
